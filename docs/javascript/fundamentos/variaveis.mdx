## Explicação Conceitual

Em JavaScript, as variáveis podem ser declaradas usando três palavras-chave: [***Var***](#var), [***Let***](#let) e [***Const***](#const).

## Var

Usada para declarar variáveis em **escopo de função**, o que significa que uma variável declarada com `var` fica acessível em toda a função onde foi declarada, independentemente de onde a declaração ocorreu dentro da função **(hoisting)**.

### Sintaxe

```js
var nome = "Kevin";
var idade = 25;
```

<Admonition type="note" title="Nota">
  Quando se declara uma variável `var` é como se sua declaração fosse movida para o topo do escopo durante a execução, mas a atribuição de valor permanece no local onde foi escrita. Também chamada popularmente de ***"Variável Global"***

  <h2>Exemplo</h2>
  <h3>Como escrevemos</h3>
  ```js
  console.log(numero); // undefined
  numero = 10;
  console.log(numero); // 10
  var numero = 5; // Declarando a variável numero e atribuindo 5 a ela
  ```
  <h3>Como o interpretador lê</h3>
  ```js
  var numero;      // A declaração de 'numero' é elevada para o topo do escopo (hoisting).
  console.log(numero); // Aqui a variável já foi declarada, mas ainda não tem valor, então é 'undefined'.
  numero = 10;     // Agora atribuímos 10 à variável.
  console.log(numero); // O valor da variável é 10.
  numero = 5;      // Agora atribuímos 5 à variável.
  ```
</Admonition>

<Admonition type="warning" title="Aviso">
  Hoje em dia, evitamos o uso de `var` porque ele tem comportamentos imprevisíveis, como <u>hoisting</u> e <u>escopo de função</u> **(não respeita o escopo de bloco)**, o que pode causar bugs difíceis de rastrear.
</Admonition>

## Let

O `let` foi introduzido no ECMAScript 2015 (ES6) e corrige vários problemas associados ao var.

A declaração `let` usada para criar variáveis com **escopo de bloco**, o que significa que a variável só é acessível dentro do bloco onde foi declarada. Diferente de `var`, `let` não permite a redeclaração da mesma variável no mesmo escopo.

### Sintaxe

```js
// Sintaxe básica do let
let nome = "Kevin"; // Declarando a variável nome e atribuindo "Kevin" a ela
let idade = 25; // Declarando a variável idade e atribuindo 25 a ela
console.log(nome); // Kevin
nome = "Kevin"; // Atribundo "Kevin" a variável nome
console.log(nome); // Kevin
```


## Const

O `const` também foi introduzida no ES6. Ela é usada para declarar variáveis que não podem ser reatribuídas após a inicialização. No entanto, isso não significa que o valor é imutável, pois **objetos** e **arrays** podem ser alterados.

### Sintaxe

```js
const nome = 'Kevin';
console.log(nome); // Kevin
```

<Admonition>
  Alguns conceitos como "objetos" e "arrays" você verá mais a frente nesse curso. Não se preocupe com esses termos agora, foque apenas em indentificar a funcionalidade das variáveis nesse momento.
</Admonition>

<Admonition type="tip" title="Dica">
  As **boas práticas** recomendam:
  - Usar `let` e `const`, e evitar usar `var` porque:
    - `let` tem escopo de bloco e não sofre de <u>hoisting</u> da mesma maneira que `var`.
    - `const` garante que o valor não será reatribuído, tornando o código mais previsível e seguro.

  - `let` vs `const`:<br />
    - Sempre que possível, use `const`, pois garante que o valor não será
  reatribuído.
    - Use `let` apenas quando precisar de uma variável cujo valor será
  alterado.

  Essas dicas tornam o código mais claro e menos propenso a erros.
</Admonition >



## Exercícios Propostos

### Exercício 1: Escopo com `var` e `let`
Crie uma variável `endereco` usando `var`. Dentro de um bloco `{ }`, declare outra variável chamada `celular` usando `let`. Em seguida, exiba ambas as variáveis com `console.log` em diferentes partes do código e observe o que acontece. Tente entender o motivo para o comportamento de cada variável.

<details>
  <summary>Resposta</summary>



```js
var endereco = "Rua A, 123"; // variável fora do bloco
{
  let celular = "123-456-7890"; // variável dentro do bloco
  console.log(celular); // Acessa 'celular' dentro do bloco
}
console.log(endereco); // Acessa 'endereco' fora do bloco
console.log(celular); // Erro: 'celular' não está acessível aqui porque é uma variável de escopo de bloco
```


</details>

### Exercício 2: Constantes com `const`
Declare uma constante chamada `pais` e atribua o nome de um país. Depois, experimente modificar o valor dessa constante e veja o que acontece. Reflita sobre por que ocorreu o comportamento observado.

<details>
  <summary>Resposta</summary>
```js
const pais = "Brasil";
pais = "Argentina";
console.log(pais);// Erro: Assignment to constant variable.
```
</details>

### Exercício 3: Reatribuindo Variáveis com `let`
Crie uma variável `cidade` e atribua o nome de uma cidade. Depois, mude o valor dessa variável para outro nome e exiba o resultado. Pense sobre o motivo pelo qual `let` permite essa alteração.

<details>
  <summary>Resposta</summary>
```js
let cidade = "Fortaleza";
cidade = "Rio de Janeiro";
console.log(cidade); // Saída: "Rio de Janeiro"
```
</details>

### Exercício 4: Diferença de Escopo Entre `var`, `let` e `const` em Blocos
Declare três variáveis: `bairro` com `var`, `rua` com `let`, e `cep` com `const`, usando algumas dentro de um bloco `{ }` e outras fora. Tente acessar essas variáveis em diferentes partes do código e observe o que acontece. Analise e reflita o motivo para a acessibilidade (ou falta dela) de cada variável.

<details>
  <summary>Resposta</summary>
```js
var bairro = "Centro";
{
  let rua = "Rua das Flores";
  const cep = "12345-678";
  console.log(bairro); // Acessível: "Centro"
  console.log(rua); // Acessível: "Rua das Flores"
  console.log(cep); // Acessível: "12345-678"
}
console.log(bairro); // Acessível: "Centro"
console.log(rua); // Erro: 'rua' não está acessível fora do bloco
console.log(cep); // Erro: 'cep' não está acessível fora do bloco
```
</details>

<Admonition type="info" title="Dica">
- **Use o Terminal**: Use o terminal do VS Code para verificar os console.log.

</Admonition>
